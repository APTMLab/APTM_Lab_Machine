1. System Overview Measurement WorkFlow Remote

Browser Interface For Scan Setup

Measurement WorkFlow Remote is the browser interface for preparing scan inputs for the Measurement WorkFlow desktop application. It is designed for phone and desktop browsers. The desktop app remains responsible for hardware communication, readiness checks, experiment execution, data saving, and plotting.

Supported acquisition families

Oscilloscope workflow

Used for Tektronix oscilloscope acquisition with B&K Precision 4055B function-generator sweep control.

Laser Doppler Vibrometer workflow

Used for PSV-680F QTec Time or FFT measurement workflows with external or PSV generator setup.

2. Requirements Measurement WorkFlow Remote

What Must Be Ready

Lab PC

  • Measurement WorkFlow desktop application installed and able to run scans locally.
  • Required instruments connected and visible to the desktop app.
  • Output folders available for saving experiment data.
  • Internet access from the lab PC when the browser website is used outside the local machine.

Website

  • A browser on a phone, tablet, or computer.
  • Website link provided by the lab administrator.
  • Lab ID.
  • Current website password.
Before running: confirm that the physical setup is safe, cables are connected correctly, the excitation path is ready, and no one is adjusting the mechanical system.
3. Login and Connection Measurement WorkFlow Remote

Sign In To The Website

The login page identifies the lab session and allows the browser to send scan setup commands to the desktop application. The browser may remember the session locally so the page can resume later.

Lab IDExample: aptm-lab

Status indicators

The website shows readiness, desktop connection, idle/running state, and selected workflow.

ReadyIdleOscilloscope

Login procedure

  1. Open the Measurement WorkFlow Remote website.
  2. Confirm the website address if the field is visible.
  3. Enter the Lab ID.
  4. Enter the website password.
  5. Select Login.

Clear Saved Login

Use Clear Saved Login when the password changes, the website address changes, or another lab member needs to log in with their current credentials.

Login duration: If the website reports an expired login, clear the saved login and sign in again.

5. Setup and Detect Measurement WorkFlow Remote

Prepare Instruments Before Entering Scan Inputs

LDV setup fields

Field Description
Output folder Folder where scan data should be saved. If left blank on the website, the default data folder is used.
Mass count Number of masses or response points expected in the LDV dataset.
Experiment settle (s) Wait time after a frequency or excitation-state change before acquisition begins.
Generator Mode External Function Generator or PSV Internal Generator. The PSV internal generator mode does not use phase input.
Scan Type Frequency/Amplitude/Phase or ε and δ0 sweep.
External Generator VISA Resources Detected or manually pasted B&K Precision 4055B VISA resource strings.

Oscilloscope setup fields

Field Description
Output folder Folder where oscilloscope scan data should be saved.
Oscilloscope VISA Resources Detected or manually pasted Tektronix oscilloscope VISA resources.
Function Generator VISA Resources Detected or manually pasted B&K Precision 4055B function-generator resources.
Generator Mode External Function Generator. Oscilloscope workflow uses external generator control.
Scan Type Frequency/Amplitude/Phase or ε and δ0 sweep.

Detect buttons

Detect Oscilloscopes
Asks the desktop app to detect Tektronix resources.
Detect Function Generators
Asks the desktop app to detect B&K Precision 4055B resources.
Detect Both
Runs oscilloscope and function-generator detection from one button.
Detection is not continuous. The website sends a detect command when you press a detect button. The desktop app performs detection and later reports the result back to the website.
6. Scan Inputs Measurement WorkFlow Remote

Enter The Scan Definition

Standard function-generator channel input

Each detected B&K Precision 4055B function generator has two channels. The website pages through function generators so only one generator is edited at a time.

Channel mode Active fields Use case
Sweep Frequency Frequency low, frequency high, frequency step, fixed amplitude, fixed phase. Frequency response measurements at a constant excitation amplitude.
Sweep Amplitude Fixed frequency, amplitude low, amplitude high, amplitude step, fixed phase. Amplitude-dependent response checks.
Sweep Phase Fixed frequency, fixed amplitude, phase low, phase high, phase step. Phase excitation studies using external function generators.
Fixed Condition Fixed frequency, fixed amplitude, fixed phase. Holding one channel constant while another channel sweeps.

Function-generator output settings

Waveform

Choose the waveform for each function generator. Default is Sine.

CH Coupling and Wave Combine

Configure output behavior when the generator should couple channels or combine waveforms.

ε and δ0 sweep input

This mode keeps the ε and δ0 logic from the desktop workflow while also allowing frequency control. If only one function generator is active, it is selected automatically. If multiple generators are active, choose the target generator.

Control Description
Amplitude cap Vpp Maximum voltage cap used by the ε/δ0 logic.
Sweep Select Sweep Frequency, Sweep ε, or Sweep δ0.
Same Frequency For CH1 And CH2 When enabled, CH2 follows CH1 frequency inputs.
Divide by step / points Choose whether the step field represents a direct step size or a number of points.
7. Run Control Measurement WorkFlow Remote

Apply, Run, Pause, Stop

The Run Control panel sends direct workflow commands to the desktop app. The desktop app still checks readiness before the experiment starts.

Button Action
Back to Setup Return to the Setup and Detect stage without running the experiment.
Refresh Requests the latest status and detected resources from the desktop app.
Apply Sends current scan inputs to the desktop app without starting the scan.
Apply & Run Sends current scan inputs and requests scan start.
Pause Requests a pause/resume style control action from the desktop app.
Stop Requests the active scan to stop.
Outputs Off Requests function-generator outputs to turn off.
Reset Inputs Requests a reset of desktop scan inputs.

Long scans

During a run, the website checks for updated status periodically. If the phone sleeps or the browser is suspended, the desktop experiment can continue. Reopen the website and log in again if needed to view updated status.

8. Transmission Spectrum View Measurement WorkFlow Remote

Review The Final Plot

After a scan completes and the desktop app reports transmission-spectrum data, the remote website can show the final transmission-spectrum figure. This view is intentionally minimal: it displays the plot only, so it can be reviewed quickly on a phone.

What is displayed

  • Amplitude response across completed scan points.
  • Phase response between masses when available.
  • Mass-driver phase when available.
  • Legend entries for reported masses or response points.
Plot source: The plot is generated from the desktop app status payload. If the desktop app has not reported a completed transmission spectrum, the website will continue to show scan controls and status instead of the final plot.
9. Troubleshooting Measurement WorkFlow Remote

Common Problems

Message or behavior Likely cause Action
Invalid login The Lab ID or website password is not correct. Confirm the current password with the lab administrator, then clear saved login and sign in again.
Failed to fetch Wrong website address, network issue, or the website service is temporarily unavailable. Confirm the website address, check internet connection, and try refreshing the page.
Desktop app is not connected The desktop app is closed or not currently accepting website commands. Open Measurement WorkFlow on the lab PC and confirm the website connection is enabled.
Detect command sent, but resources do not update Instrument is not visible to VISA, app is busy, or status has not refreshed yet. Wait for detection refresh, press Refresh, and verify resources locally on the lab PC.
Website session appears lost during a long experiment Phone browser slept or the tab was suspended. Reopen the website, log in if needed, and refresh status. The desktop scan can continue independently.
Transmission spectrum does not appear The desktop app has not completed or has not reported spectrum data. Check the desktop app scan status and confirm completed metrics exist.
